# RAG Система для Вопросов и Ответов

Система для вопросов и ответов на основе документов с использованием RAG (Retrieval-Augmented Generation).

## 📋 Что это такое?

Эта система позволяет:
- Загружать документы (PDF, DOCX)
- Извлекать текст и изображения из документов
- Индексировать контент в векторной базе данных
- Задавать вопросы по документам
- Получать ответы на основе содержимого документов

## 🚀 Быстрый старт

### Шаг 1: Установка Python

Убедитесь, что у вас установлен Python 3.8 или выше:
```bash
python --version
```

### Шаг 2: Установка зависимостей

Установите все необходимые библиотеки одной командой:
```bash
pip install -r requirements.txt
```

### Шаг 3: Установка дополнительных компонентов

#### Tesseract OCR (для извлечения текста из изображений)

**Windows:**
1. Скачайте установщик с [GitHub Releases](https://github.com/UB-Mannheim/tesseract/wiki)
2. Установите Tesseract
3. Добавьте путь к Tesseract в переменную PATH или укажите путь в коде

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get install tesseract-ocr
sudo apt-get install tesseract-ocr-rus  # для русского языка
```

**macOS:**
```bash
brew install tesseract
brew install tesseract-lang  # для поддержки языков
```

#### Qdrant (векторная база данных)

**⚠️ ВАЖНО: Qdrant необходим для работы системы!**

**Вариант 1: Docker (если установлен Docker)**
```bash
docker run -p 6333:6333 -p 6334:6334 qdrant/qdrant
```

**Вариант 2: Без Docker - скачать бинарный файл (РЕКОМЕНДУЕТСЯ)**

**Windows:**
1. Скачайте последний релиз: https://github.com/qdrant/qdrant/releases
2. Найдите файл для Windows (например, `qdrant-1.7.0-x86_64-pc-windows-msvc.zip`)
3. Распакуйте архив в любую папку (например, `C:\qdrant`)
4. Запустите `qdrant.exe` (дважды кликните или через командную строку)
5. Оставьте окно открытым!

**Linux/macOS:**
- Скачайте соответствующий релиз с GitHub
- Распакуйте и запустите бинарный файл

**Вариант 3: Облачный Qdrant (бесплатный план доступен)**
1. Зарегистрируйтесь на https://cloud.qdrant.io/
2. Создайте кластер
3. Получите URL и API ключ
4. Обновите настройки в коде (см. файл `УСТАНОВКА_QDRANT_БЕЗ_DOCKER.md`)

**📖 Подробная инструкция:** См. файл `УСТАНОВКА_QDRANT_БЕЗ_DOCKER.md`

### Шаг 4: Настройка API ключей (опционально)

Если вы хотите использовать OpenAI API, создайте файл `.env` или установите переменную окружения:
```bash
# Windows PowerShell
$env:OPENAI_API_KEY="ваш-ключ-здесь"

# Windows CMD
set OPENAI_API_KEY=ваш-ключ-здесь

# Linux/macOS
export OPENAI_API_KEY="ваш-ключ-здесь"
```

Или просто введите ключ в интерфейсе Streamlit после запуска.

### Шаг 5: Запуск приложения

Запустите главный файл приложения:
```bash
streamlit run "streamlit_app (1).py"
```

Или если у вас проблемы с пробелами в имени файла:
```bash
streamlit run streamlit_app^ ^(1^).py
```

Приложение откроется в браузере по адресу: `http://localhost:8501`

## 🎯 Как использовать

1. **Запустите приложение** (см. Шаг 5 выше)

2. **В боковой панели настройте:**
   - Тип модели (OpenAI, Ollama, или Local)
   - Название модели
   - API ключ (если используете OpenAI)

3. **Загрузите документы:**
   - Нажмите "Выберите документы для загрузки"
   - Выберите PDF или DOCX файлы
   - Дождитесь обработки

4. **Задайте вопрос:**
   - Введите вопрос в поле ввода
   - Нажмите Enter
   - Получите ответ на основе ваших документов!

## 🔧 Решение проблем

### Ошибка подключения к Qdrant
```
Ошибка: Не удалось подключиться к Qdrant
```
**Решение:** Убедитесь, что Qdrant запущен:
```bash
docker run -p 6333:6333 qdrant/qdrant
```

### Ошибка импорта модулей
**Решение:** Убедитесь, что все зависимости установлены:
```bash
pip install -r requirements.txt
```

### Tesseract не найден
**Решение:** Установите Tesseract OCR и добавьте в PATH, или укажите путь в коде:
```python
pytesseract.pytesseract.tesseract_cmd = r'C:\Program Files\Tesseract-OCR\tesseract.exe'
```

### Медленная загрузка моделей
**Решение:** При первом запуске модели загружаются из интернета. Это нормально. При последующих запусках они будут загружаться из кэша.

## 📝 Примечания

- При первом запуске система загрузит модели для эмбеддингов (может занять время)
- Изображения сохраняются в папке `temp_extracted`
- Для работы с русским языком убедитесь, что установлен `tesseract-ocr-rus`
